```
Мій Obsidian vault знаходиться в ~/MyVault з такою структурою:

📁 ~/MyVault/

## 📋 КРОК ЗА КРОКОМ:

### КРОК 1: Перевірте структуру
```bash
# Переконайтесь що папки існують:
ls -la ~/MyVault/10-Work/Projects/
ls -la ~/MyVault/20-Trading/Journal/
ls -la ~/MyVault/20-Trading/Analysis/
```

### КРОК 2: Прочитайте файли з правилами
Прочитайте три файли в корні ~/MyVault:
- `CLAUDE-RULES.md` — Повна документація правил
- `CLAUDE-SETUP.md` — Інструкція з прикладами
- `vault_writer.py` — Python скрипт для запису

### КРОК 3: Зрозумійте систему
1. Для КОЖНОЇ мої сесії, я повинен записати файл у правильну папку
2. Файл повинен мати YAML frontmatter (title, date, tags, category, status, pinecone_indexed)
3. Контент повинен слідувати шаблону (відповідь під для проектів/трейдингу)
4. Лінки повинні бути wiki-style: [[10-Work/Projects/назва]]

### КРОК 4: На кінець КОЖНОЇ сесії
Коли я (агент) закінчу роботу, я повинен:

1. **Прочитати** чим займався
2. **Видобути** ключову інформацію:
   - Виконані задачи
   - Рішення які були прийняті
   - Проблеми та їх рішення
   - Артефакти (посилання на файли, код)

3. **Написати** файл у правильну папку:
   - Ім'я файлу: `YYYY-MM-DD-[коротко-опис].md`
   - Папка залежить від типу: `10-Work/Projects/[назва]/sessions/` або `20-Trading/Journal/`
   - Контент: YAML + структурований markdown

4. **Повідомити** користувача:
   ```
   ✨ Сесія завершена. Записую в пам'ять...
   
   📝 Створено: [шлях-до-файлу]
      ├── ✅ [кількість] завдань виконано
      ├── 🔑 [кількість] рішень прийнято
      ├── 🐛 [кількість] багів вирішено
      └── 🔗 Пов'язано з [список проектів]
   ```

### КРОК 5: Тестування
Створіть тестовий файл вручну:
```bash
# Для проекту
cat > ~/MyVault/10-Work/Projects/test-project/sessions/2026-04-19-test.md << 'EOF'
---
title: "Test Session"
date: 2026-04-19
tags: [work, session, test]
category: work
project: test-project
status: completed
pinecone_indexed: false
---

## 🎯 Мета сесії
Тестування системи

## ✅ Виконано
- Test task 1
- Test task 2
EOF
```

Перевірте що файл створився та має правильний формат.
